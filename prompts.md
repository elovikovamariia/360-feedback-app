# История промптов к ИИ при разработке

Документ для сдачи по **заданию 4** (история запросов к ассистенту при реализации).  
Формат: дата → цель → суть запроса → результат в коде.

---

## 2026-04-14 — Адаптив и мобильный UX

**Цель:** единый веб-интерфейс для ПК и телефона без отдельного native-приложения.

**Запрос (суть):** mobile-first: safe-area, touch targets ~44px, таблицы со скроллом, компактные графики, порядок блоков на странице результатов, viewport и theme-color.

**Результат:** `layout.tsx`, `globals.css`, `AppHeader.tsx`, `PageChrome.tsx`, `ChartTabs.tsx`, `ResultsRadar.tsx`, `CompetencyBarCompare.tsx`, `ResultsClient.tsx`, обёртки `.table-scroll` на ключевых таблицах.

---

## 2026-04-14 — HR / Top Management: цикл и сводка по компании

**Цель:** выбор цикла, сводка по организации, доступ топ-менеджера без полной HR-панели.

**Запрос (суть):** API `allCycles`, переключатель цикла, блок «Обобщённая оценка по компании», доступ `RoleGuardAny` с `company_rollups`, ссылки с «Отчётов», крошки для executive.

**Результат:** `src/app/api/cycles/[id]/route.ts`, `HrCycleDetailPageClient.tsx`, `reports/page.tsx`, `ResultsClient.tsx`, `roles.ts`, `demo-scenario.ts`, `static-api.ts` (GitHub Pages).

---

## 2026-04-14 — Иллюстрация масштаба (~2000 оцениваемых)

**Цель:** показать вид корпоративной сводки при большой выборке (учебный макет).

**Запрос (суть):** данные и UI-блок с гистограммой, компетенциями, сегментами, темами и рисками; показывать HR и Top Management при 100% заполнения.

**Результат:** `src/lib/enterprise-scale-illustration.ts`, `EnterpriseRollupIllustration.tsx`, встраивание в `ResultsClient` и `HrCycleDetailPageClient`.

---

## 2026-04-13 — Концепция 360, тон интерфейса, компетенции

**Цель:** пять компетенций как в документе «Концепция 360°»; убрать избыточные пометки «демо» в UI и текстах.

**Запрос (суть):** Patient first, Play to win, Unite efforts, Embrace change, Less is more в сиде и UI; нейтральные формулировки вместо «тест/демо» где уместно.

**Результат:** `prisma/seed.ts`, визуалы и отчёты (`competency-visuals.ts`, `ai-report.ts`, страницы), README и сценарии.

---

## 2026-04-13 — ИИ-агент HR и отчёт v2

**Цель:** явный агент, проверка режима LLM, структурированный HR-отчёт.

**Запрос (суть):** модуль агента, `GET /api/ai/agent`, `POST` генерации отчёта, JSON-схема v2 (инсайты, текст, бенчмарки, риски, план 90 дней), локальный fallback без ключа.

**Результат:** `src/lib/hr-ai-agent.ts`, `src/lib/ai-report.ts`, `src/app/api/ai/agent/route.ts`, интеграция в страницу результатов и сид при необходимости.

---

## 2026-04-13 — Старт реализации сервиса

**Цель:** сквозной MVP по ТЗ задания 4.

**Запрос (суть):** Next.js (App Router), Prisma + SQLite, анкета по токену, HR-дашборд, радар и столбцы, AI-отчёт (OpenAI-совместимый API или локальный шаблон), сиды, README, `prompts.md`.

**Результат:** каркас репозитория `360-feedback-app`, схема БД, основные маршруты и UI.

---

## Примечание для проверяющего

Системные промпты и JSON-схема для модели задаются в коде (`src/lib/ai-report.ts`), а не в этом файле. Здесь зафиксированы **продуктовые запросы** к ассистенту при итерациях разработки.
