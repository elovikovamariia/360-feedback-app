# Тестовый сценарий: сбор 360° → завершение → отчёты и ИИ

Приложение с переключением ролей в шапке (HR / Руководитель / Сотрудник / Респондент). Ниже — «счастливый путь» для проверки UX и логики.

## Предусловия

- Выполнены `npx prisma db push` и `npm run db:seed` (или свой цикл создан из HR).
- Для **реального** ИИ в `.env` заданы `OPENAI_API_KEY`, при OpenRouter — `OPENAI_BASE_URL` и модель (см. `.env.example`).

## Шаг 1. HR запускает цикл

1. Роль **HR**.
2. Раздел **«Оценка 360»** → форма **«Запустить цикл»**: выберите полугодие, задайте **даты сбора** (начало не раньше сегодня, конец ≥ начала).
3. Отправьте форму → откроется карточка цикла.
4. Убедитесь: блок **«Процесс для HR»** (этапы 1→2→3), таблица **«Респонденты»** со ссылками **«Открыть анкету»**, у каждой анкеты видно **кого оценивают** и **роль** (самооценка / руководитель / коллега).

**Ожидание:** при запуске автоматически создаются назначения: самооценка, руководитель из `managerId`, один коллега из той же орггруппы (выбор коллеги — по алфавиту ФИО, см. `src/lib/cycle-assignments.ts`).

## Шаг 2. Сотрудник — самооценка

1. Роль **Сотрудник** (пример: **Анна Соколова**) или откройте ссылку самооценки из таблицы HR по нужному человеку.
2. Страница анкеты `/survey/{token}`:
   - сверху **«Сроки проведения оценки (сбор)»** с датами цикла;
   - заголовок **самооценки**, блок **«Вы оцениваете себя»** (не путать с оценкой коллеги).
3. Заполните шкалу 1–5 по всем компетенциям, общий комментарий ≥ 20 символов → **«Отправить анкету»**.

**Ожидание:** после отправки — блок **«Что дальше»**, ссылка на отчёт (полная сводка возможна после ответов остальных).

## Шаг 3. Коллега и руководитель

1. Роль **Руководитель** (пример: **Дмитрий Волков**) или **Респондент** (пример: **Борис Панов** — в «Мои анкеты» в блоке «Оценка других» должна быть строка на **Анну Соколову**). Можно также открыть прямые ссылки из карточки цикла HR.
2. В анкете:
   - те же **сроки сбора** вверху;
   - явный блок **«Кого вы оцениваете»** с ФИО сотрудника;
   - подпись роли (руководитель / коллега).
3. Заполните и отправьте.

**Ожидание:** после отправки у респондента — краткий баннер о фазе сбора; свои комментарии в архиве анкеты видны **оценщику**, но не сотруднику в итоговом отчёте.

## Шаг 4. HR — завершение и аналитика

1. Роль **HR** → карточка цикла.
2. Дождитесь **100%** отправленных анкет по всем назначениям (после начального сида цикл может быть уже заполнен; для нового цикла — допройти всех).
3. **Ожидание:**
   - баннер **«Цикл завершён»**;
   - кнопка якоря **«К списку оцениваемых и отчётам»**;
   - у каждого оцениваемого ссылка **«Результаты и AI»** — радар, HR-отчёт v2, бенчмарки, риски, план 90 дней;
   - при настроенном ключе — **«Перегенерировать»** вызывает LLM; `GET /api/ai/agent` → `mode: "llm"`.

## Шаг 5. Сотрудник — итог без чужих текстов

1. Роль **Сотрудник** → **«Мои результаты»** / отчёт `/results/view?revieweeId={id}&cycleId=...` (на GitHub Pages — без динамического сегмента в пути).
2. **Ожидание:**
   - видны **баллы**, радар, сравнение самооценки и окружения;
   - **нет** дословных комментариев коллег и руководителя;
   - есть поясняющий блок **«Ваш персональный отчёт»** и **HR-отчёт с ИИ** (рекомендации на основе текстов на сервере).

## Контрольные точки UX

| Экран            | Проверка |
|------------------|----------|
| Анкета           | Сроки сверху; самооценка ≠ оценка коллеги; ФИО оцениваемого для не-self |
| Мои анкеты       | У задач строка «Сбор оценок: дата — дата» |
| HR цикл          | Этапы 1–3; при 100% — CTA к отчётам |
| Отчёт сотрудника | Нет сырых чужих текстов; есть ИИ-блок |

При расхождении с ожиданиями приложите скрин и выбранную роль в шапке.
