# Сервис оценки 360° с AI-аналитикой (демо)

Веб-приложение на **Next.js 13 (App Router)**, **Prisma 5**, **SQLite** (локально), **Recharts** (радар). AI-отчёт: вызов **OpenAI Chat Completions** при наличии `OPENAI_API_KEY`, иначе — **демо-генератор** на сервере (структура отчёта та же). Стек выбран так, чтобы проект собирался на **Node.js 16.14+** (в т.ч. 18.13).

## Функции (соответствие тестовому заданию)

- Цикл оценки, назначение респондентов (самооценка, руководитель, коллеги, подчинённые).
- Анкета: шкала 1–5 по 5 компетенциям + обязательный свободный текст.
- Интерфейс оценщика по уникальной ссылке; в HR-таблице статусы «Отправлено / Ожидает», % заполнения.
- Радар по компетенциям, сравнение средних по группам (self / manager / peers / subordinates).
- AI: темы, паттерны, сильные стороны, зоны роста, противоречия (self vs others по средним), actionable рекомендации. Сырые тексты в JSON для клиента **не** отдаются — только на сервере для генерации.

## Быстрый старт

1. Установите зависимости:

```bash
npm install
```

2. Создайте файл `.env` (можно скопировать из `.env.example`):

```env
DATABASE_URL="file:./dev.db"
# OPENAI_API_KEY=sk-...
# OPENAI_MODEL=gpt-4o-mini
```

3. Примените схему и демо-данные:

```bash
npx prisma db push
npm run db:seed
```

4. Запуск:

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000) → раздел **HR / статусы** → выберите цикл → ссылки на анкеты и страницу **Результаты + AI**.

## Архитектура

- `prisma/schema.prisma` — сущности: люди, цикл, назначения с токеном приглашения, ответы (оценки + текст), кэш AI-отчёта.
- `src/app/api/*` — REST-подобные маршруты для циклов, анкеты, агрегатов и AI.
- `src/lib/summary.ts` — агрегация средних по ролям и подготовка обезличенного пакета текстов для модели.
- `src/lib/ai-report.ts` — вызов OpenAI или локальный демо-отчёт.

## Деплой

- **SQLite** на бессерверных платформах (например Vercel) **не подходит** для постоянного хранилища. Варианты: [Neon](https://neon.tech) / Supabase + смена `provider` в Prisma на `postgresql` и `DATABASE_URL`, либо PaaS с постоянным диском (Railway, Render).
- Задайте переменные окружения на хостинге: `DATABASE_URL`, при необходимости `OPENAI_API_KEY`, `OPENAI_MODEL`.
